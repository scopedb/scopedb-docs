Use these guides to connect to ScopeDB, create an event table, ingest data, run
queries, and add indexes.

## Start here

### [Connect to ScopeDB](/guides/connect-to-scopedb)

Set up your endpoint, API key, and local environment variables.

### [Quickstart](/guides/quickstart)

Create an event table, insert sample rows, and run the first ScopeQL queries.

### [Ingest events](/guides/ingest-events)

Load application events with ScopeQL transformations and data-quality checks.

### [Query events](/guides/query-events)

Inspect rows, search messages, extract fields from payloads, aggregate trends,
and join metadata.

### [Model flexible events](/guides/model-flexible-events)

Decide what belongs in typed columns and what belongs in a semi-structured
`object` column.

### [Add indexes](/guides/add-indexes)

Choose point, range, search, and materialized indexes from real query patterns.

## How to use these guides

If you are new to ScopeDB, follow the first two guides in order:

1. [Connect to ScopeDB](/guides/connect-to-scopedb).
2. [Quickstart](/guides/quickstart).

Then choose the guide for the task in front of you:

- write data producers with [Ingest events](/guides/ingest-events);
- inspect and aggregate data with [Query events](/guides/query-events);
- design flexible event tables with [Model flexible events](/guides/model-flexible-events);
- improve important query paths with [Add indexes](/guides/add-indexes).

For exact syntax, use the [Reference](/reference).
